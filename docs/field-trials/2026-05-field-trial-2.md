## Outcomes

What worked well:

- Full CRUD pattern is complete and consistent across all layers (handler → use case → repository → PDO)
- OpenAPI contract covers all 5 Note operations with correct response schemas
- `DomainExceptionHandlerInterface` keeps handlers thin — no `try/catch` boilerplate per handler
- `InMemoryNoteRepository` makes HTTP-level tests fast and self-contained
- `ListNotesHandler` validation (limit 1–100, offset ≥ 0) surfaces clean 422 Problem Details

What the LLM inferred easily:

- Pattern reuse: once GET/POST/DELETE existed, PUT followed the same structure without ambiguity
- OpenAPI schema reuse (`CreateNoteRequest` for PUT body)
- Test structure: adding update tests required no new test helpers — `InMemoryNoteRepository` and `Psr17Factory` composed cleanly

What docs or boundaries were unclear:

- **`X-Request-Id` not in logs**: The request ID is echoed in response headers but not attached to Monolog log records. Diagnosing a specific request requires correlating log timestamps manually. This is the most concrete friction.
- **No `PATCH` semantics documented**: PUT replaces the full resource. Partial update semantics are not addressed; no ADR guidance exists for future PATCH implementation.
- **`src/Example/` not linked from README**: A new developer cloning the repo may not immediately find the Note CRUD as the canonical architecture example.
