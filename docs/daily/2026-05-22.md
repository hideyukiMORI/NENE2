## NENE2 現状評価（Claude による）

### 強み

**① PSR 徹底と「魔法を使わない」方針**
`strict_types`、readonly、コンストラクタインジェクション、明示的ワイヤリング、PHPStan level 8。
フレームワークが何をしているかが読める品質の底の高さ。

**② howto × フィールドトライアルの組み合わせ**
87 本の howto ガイドはそれぞれ独立した PHPUnit テストスイート付きの参照実装が存在する。
セキュリティ cadence（3 本に 1 回脆弱性診断、4 本に 1 回攻撃試験）を機械的に維持。
「安全なパターンを教えるフレームワーク」という立ち位置が一貫している。

**③ AI-first の設計哲学**
MCP サーバー、OpenAPI 契約、型付き境界、小さなクラス。
フレームワークを「AI と一緒に使うもの」として最初から設計している稀有な PHP フレームワーク。

### 懸念点

**① フレームワーク本体が FT ループで育っていない**
FT96〜FT156 の学びがすべて howto に蓄積されて、`src/` にフィードバックされていない。
v2.0 設計（Phase 74）でどのパターンをフレームワーク本体に戻すかが問われる。

**② howto ライブラリの発見可能性**
87 ガイドは資産だが、初見ユーザーが自分のユースケースに合うガイドを探しやすいかは疑問。
Phase 73 のナビゲーション整備が急務。

**③ FT ループ高速化による密度の問題**
後半の FT はテスト数が確保されているが、パターン記述の深さが均一に保たれているか要確認。
FT153（57 テスト）と FT145（20 テスト）では実装複雑度が大きく異なる。

**④ FT プロジェクト群の孤島化**（→ `docs/todo/current.md` に検討事項として保持）
`../NENE2-FT/` の 156 本の参照実装が外部から不可視。
公開方針（GitHub 独立リポジトリ / howto 埋め込み / examples/ 集約）を Phase 73 前に決める。

### 総評

NENE2 は「完成度を高めながら自分自身を記録し続けるフレームワーク」という珍しい性格を持つ。
v1.5.90 時点で使えるものとして十分な深さがある。
次のステージは「ドキュメントで証明されたパターンがフレームワーク本体に戻ってくる」フェーズ。
Phase 74（v2.0 設計）がこれまでの蓄積の収穫地点になるかどうかが今後の評価の分かれ目。
