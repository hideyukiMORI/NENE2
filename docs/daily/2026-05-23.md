## 環境調査メモ

「Ubuntu 22→24 への引越しが必要か」という仮説を調べた。

- **結論: 不要**。PHP 8.4 は ondrej/php PPA 経由で Ubuntu 22.04 上に完全動作している
- `php` コマンドはすでに `php8.4` を指していた（`update-alternatives` 設定済み）
- Deprecated 警告の原因は PHP バージョンではなく **Composer v2.2.6（2022年版）** だった
- Ubuntu 22.04 LTS のサポートは 2027年4月まであるため移行コストをかける理由がない

「何が起きているか確認してから判断する」の教科書例になった。

## NENE2 現状評価（Claude による）

### 今日時点の強み

**① FT ループが本当に終わった**
FT001 から始まったフィールドトライアルが FT170 で一区切りを迎えた。170 本の参照実装と 90 本超の howto ガイドは、PHP API パターンの実装辞典として十分な厚みがある。

**② セキュリティ cadence が崩れなかった**
脆弱性診断・クラッカー攻撃試験を定間隔で実施し続けた結果、実際の脆弱性（FT117 prefix スキャン、FT170 int キャスト等）を複数キャッチできた。「テストがあれば安全」ではなく「攻撃者目線のテストがなければ見えない穴がある」ことが繰り返し証明された。

**③ 環境整備が後追いから先回りに変わりつつある**
今日の `new-ft.sh` と `ft-registry.md` は「困ってから直す」ではなく「次のループが始まる前に直す」対応。プロジェクトの成熟度が上がっている証左。

### 懸念点

**① howto ライブラリの発見可能性**
90 本超のガイドは資産だが、初見ユーザーが必要なガイドを見つけられるかは別問題。索引整備と VitePress サイト化が手つかずのまま。

**② FT ループの学びがフレームワーク本体に還元されていない**
howto に蓄積されたパターン（ctype_digit 検証、masked response、idempotency-key 設計など）が `src/` に反映されていない。v2.0 設計フェーズでの回収が問われる。

**③ FT プロジェクト群の孤島化**
170 本の参照実装が `../NENE2-FT/` に存在するが Packagist ユーザーからは不可視。公開方針未定のまま積み上がっている。

### 総評

FT ループという「安全な実験場」が NENE2 をここまで育てた。今日はその実験場に「次の人が使いやすい道具」を整えた日。小さな改善だが、積み重ねてきたものを使いやすくする作業は、作ること以上に価値がある場合がある。
