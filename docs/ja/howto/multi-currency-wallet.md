# ハウツー: マルチ通貨ウォレット

ユーザーごとの通貨残高と入金/出金/送金操作。
フィールドトライアル: FT198（`../NENE2-FT/walletlog/`）。VULN-A〜L セキュリティ監査を含みます。

## 主要パターン
- 残高は最小単位（セント）として保存 — float の精度問題を回避
- 自己送金はすべての DB 操作の前に拒否（422）
- 残高不足は UPDATE の前にチェック（409）
- IDOR: すべてのウォレットとトランザクションクエリに `WHERE user_id = :uid`
- 通貨は許可リストに対してバリデーション（ユーザー提供ではない）
