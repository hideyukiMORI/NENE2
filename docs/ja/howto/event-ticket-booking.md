# ハウツー: イベントチケット予約

ユーザーごとのチケット購入を持つイベントキャパシティ管理を実演します。
フィールドトライアル: FT196（`../NENE2-FT/ticketlog/`）。ATK-01〜12 クラッカー攻撃テストを含みます。

## パターンまとめ

| 関心事 | アプローチ |
|---|---|
| キャパシティ追跡 | `remaining = capacity - COUNT(tickets)` を読み込み時に計算 |
| 売り切れ | `remaining <= 0` の場合 409 Conflict |
| 重複購入 | `UNIQUE(event_id, user_id)` → 並行二重購入をキャッチ |
| IDOR キャンセル | `user_id` オーナーシップチェック → 不一致の場合 403 |
| 管理者キー | `hash_equals()` フェイルクローズ |

## ATK-01〜12 結果: 全件 PASS
